# AI 日报系统参考信源（公开版）v3.0.2

## 目的

这份文档给出一套可公开复用的参考信源框架，用于搭建：

- 个人 AI 资讯快报
- 群分享型 AI 日报
- 行业信息监测系统

这里列的是“适合接入”的来源类型和代表性站点，不包含私有配置、登录态或个人化权重。

## 选源原则

信源不是越多越好，关键是分层。

建议至少分成 4 层：

1. 高频新闻层  
负责覆盖当天最新动态，保证信息同频。

2. 官方一手层  
负责拿到正式发布、能力更新、官方博客、技术说明。

3. 工具/教程层  
负责补充真正可操作、可试用、可复现的工具和教程。

4. 商业/产业层  
负责融资、合作、组织变化、商业化、产业政策和竞争格局。

## 推荐信源结构

### 一、高频新闻层

适合 Hermes 抓取“最新发生了什么”。

代表来源：

- IT之家
- 36氪
- 钛媒体
- Wired
- The Verge
- TechCrunch

用途：

- 发现当天高频事件
- 覆盖公司动态、产品发布、融资新闻、行业事件

注意：

- 这类来源适合“发现事件”
- 不适合直接当作工具/教程来源

### 二、官方一手层

适合补强事实准确性和正式发布信息。

代表来源：

- OpenAI Blog
- Anthropic News / Blog
- Google AI Blog
- NVIDIA Blog
- Hugging Face Blog
- Meta AI Blog
- Microsoft 官方开发者博客

用途：

- 模型发布
- 平台能力更新
- SDK / API / 产品说明
- 官方路线变化

注意：

- 官方博客常混有“发布新闻稿”
- 不能默认都进入工具/教程栏目

### 三、工具 / 教程层

适合补强真正“可用”和“可操作”的内容。

代表来源：

- Hacker News（尤其 Show HN）
- GitHub Trending（AI / Agents / Developer Tools）
- Product Hunt（工具类）
- KDnuggets Tutorial
- MarkTechPost Tutorial / Hands-on
- 开发者社区博客（可按需加）

用途：

- 真工具
- 真教程
- 可安装、可接入、可试用的项目
- 工作流、how-to、最佳实践

注意：

- 工具层要优先看“是否有使用路径”
- 不是看到工具名就进工具池

### 四、商业 / 产业层

适合做企业动态、商业模式、产业判断。

代表来源：

- 36氪
- 钛媒体
- 财联社
- Bloomberg
- Financial Times
- The Information（如果可接入）
- 政策/产业协会公开信息源

用途：

- 融资
- 合作
- CEO / 组织变动
- 定价和商业化动作
- 政策与产业趋势

注意：

- 这一层适合正式日报和商业分析
- 不适合直接充当工具/教程层

## 对应到系统里的使用方式

### Hermes 适合用哪些源

Hermes 是提醒层，重点是：

- 先发现值得关注的信息
- 保持最新节奏

所以更适合接：

- 高频新闻层
- 官方一手层
- 少量高质量工具/教程层

Hermes 不需要一开始把所有深度源都接满。

### OpenClaw 适合依赖哪些源

OpenClaw 是正式输出层，重点是：

- 从候选里正式选题
- 对少量内容读原文
- 写摘要和分析

所以它更适合：

- 复用 Hermes 抓回来的候选池
- 再根据原文链接回看一手内容

OpenClaw 不需要自己额外做全量抓取。

## 针对其他行业怎么迁移

这套思路可以直接迁移到别的行业，原则不变，只替换信源。

例如：

- 消费电子：科技媒体 + 官方发布会页面 + 供应链/券商公开信息
- 出海 SaaS：产品发布平台 + SaaS 媒体 + 企业博客 + 定价页
- 医疗 AI：学术期刊 + 医疗政策来源 + 医疗器械/企业公告
- 新能源：产业协会 + 企业公告 + 财经媒体 + 技术社区

迁移时要保留的结构：

1. 高频新闻层
2. 官方一手层
3. 工具/方法层
4. 商业/产业层

改的只是来源，不是系统方法。

## 最后结论

这套系统的信源设计，不是为了“抓得越多越好”，而是为了：

- 最新信息能及时进库
- 高价值信息能被筛出来
- 正式输出层有足够可靠的原文依据

对公开复用版本来说，最重要的不是复制某个具体站点清单，而是复制这套信源分层方法。
